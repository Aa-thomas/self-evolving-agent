Model Completion Boundary

A compact reference for the object that Project 1A's loop calls as model.complete(messages).

Core Idea

The model boundary is one method: accept the current message state and return raw assistant text. The harness owns every decision after that.

assistant_output = model.complete(messages)

Contract

Part Meaning
model Any object with a .complete(messages) method.
messages The full conversation state so far, usually a list of role/content dictionaries in this project.
complete A method. It may call an API, return a fake output, or raise a clear error if misconfigured.
str The return value: raw assistant text for the loop to parse.

Class Checklist

  • Use class Name: with a colon.
  • Use def __init__(self, ...): to store configuration or fake state.
  • Store per-instance values as self.value, not as shared class variables.
  • Use def complete(self, messages): for the loop boundary.
  • Return raw text. Do not parse JSON inside this method.

Typing Options

For Project 1A, either keep the type loose while tests guide behavior, or use a small protocol once the shape stabilizes.

from typing import Protocol

class Model(Protocol):
    def complete(self, messages: list[dict[str, str]]) -> str:
        ...

A protocol says: this function accepts any object with the right method shape. It does not require the real model and fake model to inherit from the same base class.

Ownership Boundary

Belongs in model.complete Belongs in run_agent
Calling the provider or returning fake text. Parsing the text as JSON.
Remembering provider config, model name, or fake outputs. Validating tool request shape and tool args.
Returning one assistant string. Executing tools, rejecting bad requests, appending observations, and stopping on submit.
