Agent Loop Contract

A compact reference for Project 1A's `run_agent` loop.

Core Idea

The model proposes the next action. The harness decides whether that action is valid, executes allowed code, records the result as an observation, and calls the model again.

Primary source: OpenAI's function-calling guide describes the same flow as: request to model, receive tool call, execute application code, send tool output back, then receive a final response or more tool calls.

Function Contract

Part Meaning
user_task The user's goal. It becomes the initial user message.
model An object with .complete(messages). It returns raw assistant text.
tools A dictionary from tool name to callable. The loop may call only these tools.
max_steps The maximum number of model/tool cycles. It prevents infinite loops.

Minimal Algorithm

messages = [{"role": "user", "content": user_task}]
trace = []

for step in range(max_steps):
    assistant_output = model.complete(messages)
    messages.append({"role": "assistant", "content": assistant_output})

    request = parse_and_validate(assistant_output)

    if request is invalid:
        observation = rejection_result(request.error_code)
        messages.append({"role": "tool", "content": observation})
        continue

    if request.tool == "submit":
        return AgentResult(
            exit_reason="submitted",
            final_answer=request.args["answer"],
            trace=trace,
        )

    if request.tool not in tools:
        observation = unknown_tool_result(request.tool)
    else:
        observation = tools[request.tool](**request.args)

    messages.append({"role": "tool", "content": observation})

return AgentResult(
    exit_reason="max_steps",
    final_answer=None,
    trace=trace,
)

Invariants

  • The model never executes tools directly.
  • Every model output is treated as untrusted text until parsed and validated.
  • submit is a stop signal, not a normal external tool call.
  • A tool result or rejection becomes an observation message before the next model call.
  • max_steps returns a controlled exit instead of spinning forever.
