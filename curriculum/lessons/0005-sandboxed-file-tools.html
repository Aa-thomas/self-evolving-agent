Tools are permissioned code paths

read_file, write_file, and list_files are useful only because the harness restricts them to a sandbox root.

The Sandbox Rule

safe_path = (sandbox_root / user_path).resolve()
safe_path must stay inside sandbox_root

validate_path rejects empty paths, absolute paths, parent traversal, and resolved paths outside the sandbox. Tool handlers call this before touching the filesystem.

Security habit: validating a tool request schema is not enough. The tool implementation still enforces its own boundary.

The Three Tools

Tool Allowed result
read_file(path) Returns file text or a structured read rejection.
write_file(path, content) Writes text inside the sandbox and returns bytes written.
list_files(path=".") Returns sorted names in a sandbox directory.

Practice: Classify Paths

Which of these should be rejected and why: notes.txt, ../secret.txt, /tmp/secret.txt?

notes.txt is allowed if it resolves inside the sandbox. ../secret.txt is parent traversal. /tmp/secret.txt is absolute. The last two are rejected.
