Project 1A Primitive 3

A tool call starts as untrusted JSON text

Before schemas, tools, or loops, the harness must parse one assistant string into one Python value.

The Protocol

{"tool": "read_file", "args": {"path": "notes.txt"}}

The model does not call read_file. It emits text that asks for read_file. The harness parses that text with json.loads, then later decides whether the request is valid.

Separate the stages: valid JSON is not the same thing as a valid tool request. A string like "hello" is valid JSON but not a Project 1A tool request.

Where It Lives

  • 03_parse_tool_request.py: runnable parser entry point.
  • Core helper to create: parse_json(raw_text).
  • Proof to create: valid protocol JSON becomes a structured value, malformed JSON returns INVALID_JSON.

Practice: Spot The Boundary

What does primitive 3 prove, and what does it not prove yet?

It proves the assistant text can be parsed as JSON. It does not yet prove the object has the right schema, names a known tool, or has valid args.

Checkpoint

  • I can write the one-object tool JSON shape from memory.
  • I can explain why the model is only making a request.
  • I know malformed JSON must be rejected before schema checks.
