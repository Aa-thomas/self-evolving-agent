Project 1A · Primitive 4

Validate a parsed request before execution

Primitive 3 already proved that text has the outer ToolRequest shape. This primitive decides whether its tool and arguments satisfy the protocol contract.

Inspect what already exists

Read 03_parse_tool_request.py. Its ToolRequest can contain tool="read_file" and args={"path":123}; JSON and outer shape are valid. Inspect the starter target 04_validate_tool_args.py: the registry names protocol capabilities and each ToolSpec parses its own arguments.

Invariant: runtime tools receive only a typed, registered request. Parsing must not guess schemas; an individual file tool must not be the first place to discover an invalid argument type.

Commit a design decision

For ToolRequest(tool="read_file", args={"path":123}), which boundary owns the rejection?

Tool argument validation owns it and returns INVALID_TOOL_ARGS. The parser has already done its narrow job; deferring this to the runtime tool duplicates the protocol contract at the wrong boundary.

Build the target and run proof

Implement or reconstruct validate_tool_args(), then run uv run pytest curriculum/phase-1/04_validate_tool_args/validate_tool_args_tests.py. In particular, test_wrong_tool_arg_type_is_rejected proves the new gate. It does not prove that a separately configured runtime-handler dictionary contains that tool.

Choose the responsible source of truth

read_file with {"path":"notes.txt"}

delete_file with {"path":"notes.txt"}

read_file with {}

read_file with {"path":123}

read_file with path plus random

list_files with {}

Explain the remaining ambiguity

submit with an answer is protocol-valid, as test_submit_is_protocol_valid_but_returns_a_control_request shows. A later loop must still decide its control flow. Likewise, a protocol registry can validate a request while a separate runtime tools dictionary lacks its handler: that is configuration drift or an invalid state, not UNKNOWN_TOOL. Add a regression for the smallest new schema rule you introduce, then state what the validation proof does not establish.

Ask your teacher to compare your source-of-truth decision with the runtime-handler ambiguity.
