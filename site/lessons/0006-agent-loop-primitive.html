Project 1A · Reconstruction lab

Make the harness own every transition

Build and diagnose a bounded agent loop from the real parser, validator, tool registry, handlers, and tests in this repository.

Evidence packet

Starting artifacts: 03_parse_tool_request.py, 04_validate_tool_args.py, and 05_sandboxed_file_tools.py.

Target: reconstruct the narrow control-flow slice in agent_loop.py. Proof: tests_agent_loop.py.

Failure: a request can be valid against the protocol registry while its runtime handler is missing. The loop—not the model or tool—must normalize that as an observation.

Commit a prediction

The protocol registry contains read_file; the runtime tools dictionary is empty. What should happen after validation succeeds?

The registry proves protocol support; it does not prove the runtime has a handler. Dispatch owns that second check.

Inspect the real boundary

Read run_agent and locate the sequence: model output → validation → submit control → runtime handler lookup → normalized observation → next message. Compare test_unregistered_runtime_tool_is_rejected_without_execution with test_unknown_protocol_tool_is_rejected_without_execution.

They both surface UNKNOWN_TOOL, but their first responsible transition differs. One fails at protocol validation; the other fails at runtime dispatch.

Diagnose the transition, then see the feedback

Complete every case and justify each choice from the visible artifact—not from wording you remember.

Malformed JSON reaches the loop. What is the smallest safe intervention?

read_file is validated but absent from runtime handlers. Which boundary owns it?

delete_file has valid JSON and arguments. Where is the first bad transition?

Validated submit arrives while a tools dictionary also has submit. What happens?

A registered handler raises an exception. What does the loop add next?

The final allowed step does not submit. What is the next transition?

Reconstruct the narrow loop

From a skeleton, write only this slice: append assistant output; validate it; return on submit; normalize a missing runtime handler or tool exception; append one observation; respect max_steps. Do not copy the finished function.

Run uv run pytest curriculum/phase-1/06_agent_loop/tests_agent_loop.py. The proof establishes tested control-flow behavior; it does not prove every tool is correct, that the model is reliable, or that a production system has enough observability.
