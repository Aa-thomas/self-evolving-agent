Project 1A · Primitive 7

Build the causal trace the loop is missing

You will turn one agent trajectory into stable evidence without making the trace file depend on every internal runtime type.

Encounter the failure

Open partial-run.json. It says only final_answer: null and exit_reason: max_steps. Compare it with agent_loop.py: did the model repeat a valid request, did the request pipeline reject it, did dispatch fail, or did a tool raise? The final result cannot tell you.

Invariant: each step preserves the assistant output, stable request outcome, classified action, tool outcome, and exit decision. The trace describes what happened without serializing Ok, Err, Pydantic models, or other runtime implementation details.

Commit your trace contract

A trace has exit_reason="max_steps" but no assistant outputs. What exact diagnosis is impossible?

Without assistant output you cannot distinguish repetition, malformed JSON, or a changed request. Timing and storage may matter later, but neither reveals the first bad transition.

Keep the runtime and trace models separate

The runtime pipeline returns Result[ValidatedToolRequest, typed errors]. That type exists to control safe execution. A persisted trace has a different job: remain readable after runtime types are refactored.

runtime Result
    → request_result_to_trace
RequestAccepted | RequestRejected
    → TraceStep
versioned JSON

Inspect the supplied types and adapter in trace_logger.py. A rejection records the first failed RequestStage; acceptance implies every earlier request stage succeeded. This preserves causality without repeating separate {"ok": true} dictionaries.

Tradeoff: the adapter adds a small mapping layer. In return, renaming a Pydantic class or changing the internal Result implementation does not silently rewrite the trace-file contract.

Inspect the starter, then implement it

Add the narrow trace hook to run_agent, then implement TraceLogger.start, record_step, finish, and write_json. Construct the supplied trace variants at the loop boundary where each fact becomes authoritative. Do not persist raw runtime objects or reconstruct a story from the final result.

Keep the logger's working lists mutable while events arrive. In finish, freeze the completed AgentTrace with tuples. Serialize the discriminating kind fields and schema_version exactly as required by the proof.

Run uv run pytest curriculum/phase-1/07_trace_logger/tests_trace_logger.py. The adapter tests already pass. The trajectory test begins red because run_agent does not yet accept the trace hook.

Run proof and explain its limit

When the test turns green, inspect the JSON you wrote. Explain one trajectory as assistant output → request outcome → action → tool outcome or exit. Identify the first bad transition and the boundary that owns it.

The proof establishes that one covered run crosses the runtime-to-trace boundary, preserves typed causal outcomes, freezes its completed snapshot, and writes versioned JSON. It does not prove redaction, retention, backward-compatible migrations, or that every future loop branch records an event.

For transfer, design the smallest regression for a request that passes protocol validation but has no runtime handler. Which trace variant and error code should make that failure distinguishable from an unknown protocol tool?
