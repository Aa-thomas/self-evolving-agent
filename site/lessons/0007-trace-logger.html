Project 1A · Primitive 7

Build the causal trace the loop is missing

You will implement the trace logger yourself. The repository gives you a real incomplete run, a starter interface, and a proof that currently fails until your logger produces replayable evidence.

Encounter the failure

Open partial-run.json. It says only final_answer: null and exit_reason: max_steps. Compare it with the prior loop in agent_loop.py: did the model emit malformed output, did validation reject it, did a runtime handler disappear, or did the loop repeat a valid request? The result cannot answer.

Invariant: each trace step preserves the assistant output, parse result, validation result, runtime-handler decision, tool result, and exit reason. The trace is causal evidence, not a final-status log.

Commit your trace contract

A trace has exit_reason="max_steps" but no assistant outputs. What exact diagnosis is impossible?

Without assistant output you cannot distinguish model repetition, malformed JSON, or a changed request. Timing and storage may matter later, but neither explains the first bad transition.

Inspect the starter, then implement it

Read the interfaces in trace_logger.py and the previous loop in agent_loop.py. Do not look for a completed implementation: add the narrow trace hook to run_agent, then implement TraceLogger.start, record_step, finish, and write_json. Preserve the fields in TraceStep; use only JSON-safe values. Run uv run pytest curriculum/phase-1/07_trace_logger/tests_trace_logger.py. It begins red because test_trace_logger_records_causal_fields_and_writes_replayable_json runs a real agent trajectory through your logger.

Name the first missing causal field

Only the final answer and exit reason are saved.

Assistant output is saved, but the next record jumps to a tool observation.

The JSON parsed into read_file, but no record says whether its arguments were accepted.

The request validated, but the tool never ran.

The handler was selected, but the next model call repeats the request.

All per-step fields exist, but the trace ends without a run outcome.

Run proof and explain its limit

When the test turns green, inspect the JSON you wrote. Explain the first bad transition in one trace and why the responsible boundary—not “the model being weird”—owns the correction. The proof establishes that the logger preserves the supplied causal fields and writes replayable JSON. It does not yet prove redaction, retention, or that every future loop branch calls your logger. For transfer, design the smallest regression for a missing runtime handler whose protocol validation succeeded.

Ask your teacher to review the trace you produced before treating the lesson as learned.
