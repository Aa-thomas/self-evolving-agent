Project 1A Primitive 1

A model call is text in, text out, plus evidence

The first primitive does not know about tools. It only turns one prompt into one assistant string and records what the call cost in time and tokens.

The Contract

record = call_model(prompt="ping", model=model)

record.output_text
record.latency_seconds
record.prompt_tokens or record.estimated_total_tokens

The model object only needs .complete(messages). In your implementation, build a one-message conversation, time the call, print the assistant text, and log provider usage when available.

Boundary rule: model code produces text. Harness code later decides whether that text is a tool request, a rejection, or a final answer.

Implementation Target

  • 01_model_call.py: runnable entry point to review and evolve.
  • Proof to create: one prompt produces one assistant string.
  • Log to create: latency plus provider token usage or a clear estimate.

Practice: Retrieve The Shape

Without looking above, name the three things a Project 1A model-call log must capture.

Capture the assistant output text, latency, and token usage or token estimate. Cost can be estimated when pricing is known.

Checkpoint

  • I can explain why complete returns raw text.
  • I can find the latency and token fields in ModelCallRecord.
  • I know that parsing does not belong in primitive 1.
