Project 1A · Primitive 5

Make valid file requests safe to execute

Protocol validation can prove that path is a string. Only the file-tool boundary can prove that it stays inside this run’s sandbox and produces a useful observation.

Inspect the boundary you are about to change

Primitive 4 may produce a typed read_file request. Inspect 05_sandboxed_file_tools.py: validate_path() resolves a user path against sandbox_root before any read, write, or listing occurs. The target is not a generic “safe path” claim; it is a concrete result code and no outside filesystem access.

Invariant: every user path is sandbox-relative and containment is established before filesystem effects. Invalid requests return observations such as FORBIDDEN_PATH, not uncaught exceptions.

Commit a safety prediction

A validated request asks to read ../secret.txt. What should own the rejection?

The file-tool boundary returns FORBIDDEN_PATH. Primitive 4 established argument type and tool identity, but it cannot know this run’s sandbox root or filesystem state.

Implement, then produce evidence

Implement or reconstruct the path gate and file operations in 05_sandboxed_file_tools.py. Run uv run pytest curriculum/phase-1/05_sandboxed_file_tools/tests_05_sandboxed_file_tools.py. test_read_forbidden_parent_path_rejected is the regression for traversal. The proof establishes containment behavior for its fixtures; it does not prove every OS-level permission or symlink policy.

Diagnose the first file-tool decision

read_file("")

read an absolute path outside the sandbox

read_file("../secret.txt")

read notes/today.txt inside the sandbox

read missing notes.txt inside the sandbox

pass a contained directory to read_file

read a contained file larger than the byte limit

write file content to an existing contained directory

Regression and transfer

Choose one rejection and add or explain its regression test. Then inspect the resulting Ok or Err value: what exact observation would the agent loop receive? For transfer, decide whether a missing parent directory for write_file should be auto-created or return NOT_A_DIRECTORY; state the smallest safe intervention and which test would distinguish it.

Ask your teacher to review the path boundary and what the fixture-based proof still cannot establish.
