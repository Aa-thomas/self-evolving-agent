Project 1A Primitive 2

Memory is the context you pass back in

There is no hidden memory primitive in Project 1A. The model sees only the message list your harness sends on the next call.

The Contract

state = MessageState()
state.add_system("Rules")
state.add_user("Task")
state.add_assistant(raw_output)
state.add_tool(observation_json, tool_name="read_file")

messages = state.as_list()

MessageState stores four roles: system, user, assistant, and tool. When run_agent calls model.complete(messages), that list is the model's working memory.

Why Tool Messages Matter

After a tool runs, the result becomes a tool message. The next model call can react to the observation because it is now part of the passed context.

{"role": "tool", "name": "read_file", "content": "{\"ok\": true, \"value\": \"hello\"}"}

Your proof should check the second model call. It should receive the earlier assistant output and the tool observation in the message list.

Practice: Explain Memory

In one sentence, explain why the second model call can know the first tool result.

The harness appends the tool result as a tool message, then passes the full message context into the next model.complete call.

Checkpoint

  • I can list the four message roles.
  • I can explain why memory is not magic storage.
  • I can point to the tool observation in a later model call.
