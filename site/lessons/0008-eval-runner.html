Evals define what working means

The eval runner turns local tasks into pass/fail evidence and keeps the trace for every result.

The Contract

cases = load_eval_cases("local_tasks.json")
summary = run_eval_suite(cases, run_root="traces/evals/project_1a")

summary.total

Your implementation target is the eighth primitive, 08_eval_runner.py. The local task file you create should cover deterministic tasks such as direct submit, read, write, list, forbidden path, malformed JSON, unknown tool, missing file, copy file, and max-step guard.

What A Case Proves

An eval case combines a user task, scripted model outputs, optional sandbox setup, expected final answer or exit reason, and expected file contents. This keeps Project 1A honest: the loop must work in context, not only as isolated unit functions.

Practice: Unit Test Or Eval?

Why is copy_file an eval case instead of only a unit test for write_file?

It exercises the composed loop: read a file, append the observation, write another file, submit, grade the final answer, and keep the trace.

Checkpoint

  • I can distinguish a unit proof from an eval proof.
  • I know how to find the trace path for a failed eval.
